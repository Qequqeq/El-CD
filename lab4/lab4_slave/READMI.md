# SPI-слейв (ведомое устройство)

## Описание
Модуль `top_slave` реализует ведомое устройство SPI. Он принимает 8‑битные байты по протоколу SPI (режим 0, MSB first) и отображает их в шестнадцатеричном формате на двух семисегментных индикаторах. Для отладки три светодиода показывают состояние линий `CS`, `SCLK` и `MOSI`.

## Цель
Разработать SPI-слейв с синхронизацией входных сигналов, детекцией фронтов и мультиплексированием индикаторов.

## Файлы проекта
- **`top_slave.v`** -- верхний модуль: подключение SPI-слейва, управление индикаторами.
- **`spi_slave_simple.v`** -- параметризуемый SPI-слейв с синхронизацией.
- **`hex_to_7seg.v`** -- дешифратор hex‑полубайта в 7‑сегментный код.
- **`top_tb.v`** -- тестбенч для симуляции (эмулирует SPI-мастера).
- **`lab4_slave.cst`** -- файл ограничений (назначение выводов).

## Инструкция по воспроизведению
1. Установите **GOWIN EDA**.
2. Создайте проект для устройства **GW1NR9-C**.
3. Добавьте файлы `top_slave.v`, `spi_slave_simple.v`, `hex_to_7seg.v`.
4. Импортируйте `lab4_slave.cst` (или назначьте пины вручную).
5. Выполните синтез, размещение, трассировку и сгенерируйте битовый файл.
6. Прошейте плату через программатор GOWIN или `openFPGALoader`.

## Быстрый запуск
```bash
openFPGALoader -m -b tangnano9k lab4_slave.fs
```
