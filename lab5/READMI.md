# Лабораторная работа №5 – Хаотическая система Спротта с фиксированной точкой и передачей по UART

## Описание
В пятой лабораторной работе на ПЛИС **Tang Nano 9K** (GW1NR-9) реализована численная модель **хаотической динамической системы** -- уравнений Спротта для случая **R**. Используется арифметика с фиксированной точкой **Q16.16** (16 бит целой и 16 бит дробной части). Решение системы дифференциальных уравнений выполняется дискретным **методом Эйлера** с постоянным шагом. Полученные значения переменных `x`, `y`, `z` преобразуются в ASCII‑строки и передаются на компьютер через интерфейс **UART** (скорость 115200 бод, 8N1).

## Цель
Освоить реализацию численного решения ОДУ на ПЛИС с использованием арифметики с фиксированной точкой, а также организацию последовательной передачи данных по протоколу UART.

## Файлы проекта

### Основные модули
- **`ode_solver.v`** – решатель системы уравнений Спротта (метод Эйлера, Q16.16).
- **`uart_tx.v`** – параметризуемый UART-передатчик (8 бит, 1 стоп‑бит, без контроля чётности).
- **`binary_to_ascii.v`** – преобразователь 8‑битного числа в три ASCII‑цифры (сотни, десятки, единицы).
- **`freq_divider.v`** – простой делитель тактовой частоты для генерации импульсов `enable`.
- **`top_module.v`** – главный модуль, объединяющий все компоненты и управляющий процессом отправки данных.

### Тестбенчи
- **`solver_tb.v`** – тестбенч для решателя (сохраняет результаты в `output.txt`).
- **`uart_tb.v`** – тестбенч для UART-передатчика.
- **`bin_to_ascii_tb.v`** – тестбенч для преобразователя `binary_to_ascii`.
- **`freq_divider_tb.v`** – тестбенч для делителя частоты.

### Вспомогательные скрипты
- **`lab5_vizualization.py`** – скрипт на Python для преобразования двоичных данных в числа с плавающей точкой и построения графиков.
- **`generate_test_to_bin_to_ascii.py`** – скрипт для генерации тестовых векторов (файл `ASCIItest.tv`) для проверки модуля `binary_to_ascii`.

### Файлы с графиками
Результаты визуализации (временные ряды, фазовые портреты, 3D‑траектория) находятся в папке `visualization/` (см. отдельный README).

## Система уравнений (Спротт R)

$$
\begin{cases}
	x' = A - y \\
	y' = B + z \\
	z' = xy - z
\end{cases}
\quad\text{с параметрами } A = 0.9,\; B = 0.4.
$$

Метод Эйлера с шагом $h$:

$$
\begin{cases}
x_{n+1} = x\_n + h \cdot x' \\
y_{n+1} = y\_n + h \cdot y' \\
z_{n+1} = z\_n + h \cdot z'
\end{cases}
$$

Начальные условия: $x\_0 = -0.444,\; y\_0 = 1.111,\; z\_0 = -0.4$.

### Представление чисел в формате Q16.16
- 32‑битное знаковое число: биты [31:16] – целая часть, [15:0] – дробная.
- Множитель: $2^{16} = 65536$.
- Пример: $0.9$ в Q16.16 = $0.9 \times 65536 \approx 58982$ → `32'h0000E666`.
- Шаг интегрирования `STEP = 32'h000000A3` ≈ $163/65536 \approx 0.002487$.

## Инструкция по воспроизведению
1. Установите GOWIN EDA (версия 1.9.11 или новее).
2. Создайте проект для устройства GW1NR9-C.
3. Добавьте в проект все файлы .v (кроме тестбенчей, если не нужны).
4. Импортируйте файл ограничений lab5.cst или назначьте пины вручную.
5. Выполните синтез, размещение, трассировку и сгенерируйте битовый файл.
6. Прошейте плату через программатор GOWIN или openFPGALoader:
```bash
openFPGALoader -m -b tangnano9k lab5.fs
```
7. Откройте, например, плоттер в ArduinoIDE (115200 бод, 8 бит, 1 стоп‑бит, без чётности). Наблюдайте поток чисел (от 0 до 255), соответствующих значению z\_scaled (в текущей версии top\_module.v выводится z, а не x – см. строчку assign x\_scaled = (z[31:16] + 8'd128) & 8'hFF;).

## Для визуализации:
1. Запустите симуляцию тестбенча solver\_tb.v в GOWIN или iverilog, чтобы получить output.txt.
2. Выполните Python‑скрипт lab5\_vizualization.py, указав в нём путь к output.txt. Скрипт построит временные ряды, фазовые портреты и 3D‑траекторию.
