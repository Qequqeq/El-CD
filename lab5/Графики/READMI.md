# Визуализация результатов лабораторной работы №5

В этой папке находятся графики, построенные по данным, полученным в ходе моделирования хаотической системы Спротта (вариант **R**) на ПЛИС.

## Источник данных

1. **Симуляция в среде Verilog** – тестбенч `solver_tb.v` (из проекта `lab5`) запускает решатель ОДУ на 500 000 тактов. При каждом обновлении переменной `x` в файл `output.txt` записывается строка:
<32‑битное двоичное x>;<32‑битное двоичное y>;<32‑битное двоичное z>;<32‑битное двоичное время>
Все числа представлены в формате **Q16.16** (16 бит целой, 16 бит дробной части, знаковое).

2. **Реальная работа ПЛИС** – прошивка отправляет по UART (115200 бод) масштабированное значение `x_scaled` (0…255) в виде трёх ASCII‑цифр, за которыми следуют `\r\n`. Эти данные можно наблюдать в плоттере Arduino IDE или любом терминале.

## Содержимое папки
- `ВременнойРяд_x(t).png` -- временная зависимость переменной `x`.
- `ВременнойРяд_y(t).png` -- временная зависимость переменной `y`.
- `ВременнойРяд_z(t).png` -- временная зависимость переменной `z`.
- `ФазовыйПортрет_x_y.png` -- фазовый портрет в плоскости `(x, y)`.
- `ФазовыйПортрет_x_z.png` -- фазовый портрет в плоскости `(x, z)`.
- `ФазовыйПортрет_y_z.png` -- фазовый портрет в плоскости `(y, z)`.
- `3D_trajectory.png` – трёхмерная траектория `(x, y, z)` с цветовой кодировкой времени.

## Обработка данных и построение графиков

Скрипт **`lab5_vizualization.py`** (находится в корне проекта `lab5/`) выполняет следующие действия:

- Читает `output.txt`, сгенерированный тестбенчем.
- Преобразует двоичные строки в числа с плавающей точкой (функция `q16_16_to_float`).
- Строит временные ряды, фазовые портреты и 3D‑траекторию с помощью библиотек `matplotlib` и `numpy`.

## Примечание
Для воспроизведения визуализации необходимо сначала запустить симуляцию тестбенча в среде, поддерживающей $fdisplay (GOWIN EDA, Icarus Verilog и др.), чтобы получить файл output.txt. Если вы хотите строить графики в реальном времени по данным с UART, потребуется модифицировать скрипт для чтения из последовательного порта (например, с помощью pyserial).
